You want to deploy a coding agent at your company. You evaluate a few options, pick one, set it up. It works. Then someone asks: "what happens when the agent runs rm -rf /?" And you realise you have no sandboxing.

Deploying agents in production is four problems, not one. And only one of those layers is genuinely mature.

Layer 1: Sandboxing — the only mature layer

The good news. E2B (92/100) provides cloud sandboxes via SDK, self-hostable. Alibaba's OpenSandbox (87/100) is the Kubernetes-native option with gVisor, Kata, and Firecracker runtimes. Both production-grade.

For local development, nono uses kernel-level enforcement with a credential proxy that keeps API keys entirely outside the sandbox. The Rust-based contenders — boxlite (micro-VMs without a daemon), zeroboot (sub-millisecond Firecracker restore), sandlock (Landlock + seccomp) — are pushing the boundaries.

If you only get one layer right, get this one.

Layer 2: Governance — the critical gap

Sandboxing prevents accidental damage. Governance prevents unauthorized actions. What data can this agent access? Who approved this workflow? Where's the audit trail?

This layer is alarmingly immature. Microsoft's governance toolkit (61/100) is the most credible entry. sovereign-shield and intentshield tackle policy enforcement — right ideas, single-maintainer projects. If you need governance today, you're building it yourself.

Layer 3: Orchestration — mature but fragmented

Multiple agents cooperating on tasks. OpenAI's SDK (98/100) is simplest. CrewAI (97/100, 45K+ stars) for multi-agent teams. Composio (94/100) for tool integrations. High quality scores across the board, but the ecosystem fragments weekly.

Layer 4: The agent itself — least important decision

Here's the thing nobody mentions: the agent is the least important architectural choice. Aider (79/100, 41K stars) is established. The "vibe coding" category is emerging fast. But any competent agent produces reasonable code. An excellent agent without sandboxing is a liability. A mediocre agent with proper sandboxing and governance is enterprise-ready.

The maturity gap

Sandboxing: Production-ready. Pick E2B or OpenSandbox. Orchestration: Mature but still settling. The agent: Works fine. Governance: Early. If your compliance team asks about agent authorization, you don't have a good answer yet.

Start with sandboxing. Then orchestration. Then agents. Watch governance — expect tooling to mature over the next 12 months.
