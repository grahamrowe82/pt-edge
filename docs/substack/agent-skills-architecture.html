"Agent skills" is the new vocabulary. Not just tool calling or function invocation — composable capabilities that agents acquire, combine, and reuse. PT-Edge tracks 2,700+ repos building this layer. Here's the map.

Skills vs tools vs MCP

Function calling is the primitive (one action, one response). MCP is the transport (standard protocol for discovering tools). Skills are higher-level: bundles of related capabilities with state and multi-step logic. A "research skill" combines web search, parsing, summarisation, and citation into one composable competency.

Where skills run: the platforms

Composio (87/100, 27K+ stars) is the quality leader — 1,000+ toolkits with built-in search, auth, and sandboxing. agno (38K+ stars) has the largest user base. AgentScope (83/100) focuses on agents you can see and trust.

Where skills are found: the collections

antigravity-awesome-skills (23K+ stars) is the largest: 1,000+ skills for Claude Code, Antigravity, and Cursor. awesome-agent-skills (11K+ stars) offers 500+ cross-agent skills.

Collections solve discovery but not quality. Not every skill in a 1,000-item list is production-ready.

The registry problem

292 registry repos, average quality 22.9/100. The ecosystem wants a universal registry but nobody has won. Acontext (3.2K stars) treats skills as a memory layer — contextual retrieval instead of static lookup. Whoever cracks skill discovery becomes the npm of agent skills.

The trust gap

Skills execute with agent-level permissions. HN captured the concern: "OpenClaw is basically a cascade of LLMs in prime position to mess stuff up" (110 points). The OpenClaw skill-integrations category averages just 14.8/100 quality — most individual skills are trivial wrappers.

The 64 repos in agent-skill-security are building trust infrastructure, but it's early. Curated collections and sandboxing help. A systematic approach to skill vetting doesn't exist yet.

Three patterns to watch

Collections outpace quality controls. 1,000+ skill collections exist alongside no automated quality vetting. Benchmarking (SkillsBench, 364 HN points) is the emerging answer.

Registries fragment before consolidating. 292 registry repos is too many. MCP handles tool discovery; skill-level discovery is still unsolved.

Trust is the binding constraint. Until skill security vetting is automated, adoption will be limited to curated, trusted sources.
