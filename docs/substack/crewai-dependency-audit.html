Ask an AI about crewAI's dependencies and you'll get: "LiteLLM is a good choice, Pydantic is standard." That's recognition, not insight. We ran crewAI's 47 dependencies against 220K+ AI repos scored daily. Here's what the data reveals.

The headline: 6 of 7 scored deps are #1 in their category

Of 47 dependencies, 8 matched our AI taxonomy. The story they tell is striking: crewAI's team didn't just pick popular libraries. They picked the quality leaders.

LiteLLM (98/100, #1 of 285 LLM API gateways). The caveat: 2,399 commits last month — roughly 80/day. That's not just "well-maintained." It's evolving fast enough that breaking changes are a real concern. Pin your versions.

ChromaDB (94/100) and LanceDB (94/100) — both #1 in their categories. crewAI defaults to LanceDB (embedded, local-first) rather than requiring a database server. Smart default for zero-infrastructure agent memory.

tokenizers (90/100, #1 of 26). qdrant-client (86/100, #2 of 51). json_repair (75/100, #1 of 54) — the dependency nobody mentions that fixes broken JSON from LLM outputs. Quietly essential.

The one category that hasn't settled: agent memory

mem0 (72/100, 49K stars) is a strong project — the most adopted agent memory tool in the ecosystem. But it sits in a category of 977 repos where approaches range from knowledge graphs (cognee, 90/100) to episodic memory to video-based retrieval. These aren't direct competitors — cognee builds structured knowledge from documents, mem0 handles conversational memory. They're more complementary than interchangeable.

The real observation: agent memory is the most volatile part of the AI stack. LiteLLM's position as LLM gateway is secure. Chroma and LanceDB are established. But memory hasn't settled — expect crewAI's memory integrations to evolve as the category matures.

The ecosystem importance gap

The five most systemically important packages in crewAI's stack — the ones depended on by the most AI projects — are all ones we don't yet score:

pydantic (1,029 AI projects depend on it), openai (866), httpx (579), mcp (421), anthropic (229).

PT-Edge tracks 220K+ AI tools but not yet the foundational plumbing underneath them. We're adding this layer — it's the most important coverage improvement on our roadmap.

What this means
