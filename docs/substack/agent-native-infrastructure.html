Your AI agent needs to send an email. Gmail blocks automation. It needs API credentials that persist between sessions. OAuth assumes a human. It needs a phone number. Twilio wasn't built for autonomous callers.

Every piece of infrastructure your agent touches was designed for humans and grudgingly tolerates machines. PT-Edge tracks 30+ repos building what we're calling agent-native infrastructure — services designed from the ground up for AI agents as first-class entities. They're scattered across 25 different subcategories because nobody has named this category yet.

Five layers forming

Identity. AgentField (881 stars) treats agents like microservices — "scalable, observable, and identity-aware from day one." GitClaw puts agent identity in git — rules, memory, and skills as version-controlled files. AiXYZ goes furthest — on-chain identity via ERC-8004 with native payment wiring.

Communication. AgenticMail gives agents their own email addresses and phone numbers — send and receive real messages without borrowing a human's inbox. MCP Agent Mail (1,800+ stars) provides async coordination with persistent inboxes, searchable threads, and file leases.

Gateways. ClawRouter (5,400+ stars) is explicitly "agent-native" — routing across 41+ models with sub-millisecond latency and USDC payments on Base and Solana. The payment layer is built in, not bolted on.

Runtime. AgentSystems is a self-hosted app store for agents with container isolation and credential injection. Jentic Mini sits between your agent and the outside world — "your agent says what it wants to do, Jentic handles the how."

Governance. Microsoft's Agent Governance Toolkit brings zero-trust identity and policy enforcement specifically for agents.

25 subcategories and no name

The most telling signal: these repos are scattered across 25 different subcategories. Agent payment protocols, MCP gateway infrastructure, agent security hardening, email MCP servers, lightweight agent runtimes — the taxonomy doesn't have a bucket for "infrastructure for agents as entities." That's because the category is forming right now.

On Hacker News, the pace is accelerating. AgentMail launched in January (169 points). "Agent Passport — OAuth-like identity for agents" appeared in February. "KeyID — email and phone infrastructure for AI agents" in March. Each month brings new infrastructure designed specifically for agents as first-class entities.

Where this is heading

As agents become more autonomous, they need more of their own infrastructure. Today it's email and credentials. Tomorrow it's payment rails, scheduling, persistent storage, social accounts. Every service in the human tech stack will eventually have an agent-native equivalent.
