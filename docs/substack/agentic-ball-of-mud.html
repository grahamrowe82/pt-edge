Most agentic AI systems today have the same fundamental architecture as a badly written cron job: a single ephemeral process, holding all state in local memory, executing steps sequentially, with no durable record of what it did or why.

The context window is the "memory." The LLM's next-token prediction is the "orchestrator." If the process crashes, you start from scratch. Backend engineering identified and solved every one of these problems over the last two decades. The agent ecosystem is rediscovering them — slowly, out of order, and mostly by reinventing existing infrastructure badly.

PT-Edge tracks 24,000+ repos in the agents domain. We mapped them against four architectural problems to find who's building real solutions — and where the gaps are.

Memory: active but bolted on

The most actively addressed problem. mem0 (52,000+ stars, 2.8M downloads/month) is integrated into CrewAI, Agno, AgentScope, and Camel. Cognee (quality 80/100) has the highest development velocity — 372 commits in 30 days.

But memory is a plugin, not a foundation. In CrewAI's dependency list, mem0ai sits alongside chromadb as an optional add-on. The context window is still the default state store. AgentState gets this right — WAL+snapshots, CRDTs, and idempotency guarantees. The gap between "memory plugin" and "durable state layer" is where the real engineering lives.

Orchestration: the best solutions come from outside

trigger.dev (14,000 stars, 768K downloads/month) is a background jobs platform that agent developers are adopting. dagu (quality 70/100) is a declarative workflow engine from the data engineering world. Netflix Maestro is production-grade orchestration that predates the agent era entirely.

Composio's agent-orchestrator (4,300 stars, 445 commits/30d) is the standout agent-native entry — DAG-based planning, parallel agents, git worktrees for isolation. This is what "agent orchestration" should look like: a worker pulling tasks from a queue, not a prompt hoping for the best.

Observability: emerging fast

When agents run 6–8 hour tasks, stdout stops being acceptable. Cozeloop (5,400 stars) from ByteDance provides full-lifecycle management. AgentOps (5,400 stars) integrates with CrewAI, Agno, and the OpenAI SDK. agent-trace describes itself as "strace for AI agents" — the right metaphor.

February 2026 saw 14 new observability repos in a single month, up from 0–4 in prior months. The pain from long-running tasks is making visibility non-optional.

Crash recovery: the void

If step 7 of 12 fails, you rerun 1–12. Backend engineering solved this decades ago — Temporal, Inngest, DBOS, Restate are proven durable execution runtimes. The agent ecosystem has almost completely ignored them.

The dependency gap is stark: 273 repos depend on LangChain. 133 depend on ChromaDB. 2 total repos depend on any durable execution runtime. The infrastructure that prevents crashes and guarantees exactly-once execution has near-zero penetration.

SafeAgent targets exactly-once execution. DuraLang makes agents durable with one decorator. Both are under 10 stars. The first bridge appeared in March 2026: LlamaIndex announced DBOS integration for durable agent workflows.

Where this is heading

The agent ecosystem is solving these problems in the wrong order — from the easiest (memory) to the hardest (crash recovery). The projects that will define the next generation of agent infrastructure are the ones bridging the gap: trigger.dev, Sayiir ("simplified Temporal" in Rust), Stabilize (queue-based state machine), and the DBOS+LlamaIndex integration.

Amazon's Kiro agent deleted and recreated environments, causing a 13-hour AWS outage. Sentrial (YC W26) raised money to catch agent failures. When Y Combinator is funding the observability gap, it's real.
